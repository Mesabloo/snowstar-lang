> Working with registers:
There are 5 registers in Snow*: nr dr tr pr cr. Each one has a specific purpose:  nr stands for Null Register. It is empty and must remain empty during the whole program runtime. It may be used to check the size of a registry (comparing to sizes for example with len).  dr stands for Data Register. It contains persistent value storage, to emulate variables using addresses. It may be used to store something until the end of the program.  tr stands for Temporary Register. It can contain only one value marked as temporary. This register is used as a return register in many instructions.  pr stands for Parameters Register. It is used by some instruction (sys for instance). All values must be pushed in reverse order in it.  cr stands for Conditional Register. It is used when using the instruction cmp but it can also be used as a basic register. Some of these registers are called Stack registers (tr pr cr). If you don't know what is a stack, please read thishttps://en.wikipedia.org/wiki/Stack_(abstract_data_type). There are 3 ways of adding data inside those registers.  For pr cr, you have to use push. (Link to pushhttps://mesabloo.github.io/snowstar-lang/reference.html#push)  For dr, you have to use put. (Link to puthttps://mesabloo.github.io/snowstar-lang/reference.html#put)  For tr, you have to use new. (Link to newhttps://mesabloo.github.io/snowstar-lang/reference.html#new)
> The basic syntax:
Snow* is written under a static syntax. Each line of code follows the same rules:
instruction [arg0, arg1...] [% comment]  >[] stands for unrequired instruction argument. #* It is possible to put a multiline comment wherever in a line, as long as it is properly closed. *#
Not following this syntax will result in compilation problems !
